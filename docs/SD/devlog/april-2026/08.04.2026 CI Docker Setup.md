# CI + Docker Build Setup

## Compiler Requirements

C++23 `std::print` requires **gcc-14**. ubuntu:24.04 ships with gcc-13 which lacks support. Debian trixie ships with gcc-14 natively.

CMake uses `/usr/bin/c++` not `g++`, so on ubuntu we needed symlinks. Debian trixie avoids this since its `g++` IS gcc-14.

## Dockerfile

```dockerfile
FROM debian:trixie

RUN apt-get update && apt-get install -y \
    cmake \
    ninja-build \
    g++ \
    vulkan-tools \
    libglfw3-dev \
    libvulkan-dev \
    libxkbcommon-dev \
    libxinerama-dev \
    libxcursor-dev \
    libxrandr-dev \
    libxi-dev \
    pkg-config \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /src
```

Key packages:
- `g++` — pulls gcc-14 automatically on debian trixie
- `pkg-config` — required by GLFW's CMake configuration
- X11 libs — libxkbcommon, libxinerama, libxcursor, libxrandr, libxi

## Final Workflow

```yaml
- name: Configure and build tests
  run: docker run --rm -v $(pwd):/src sd-build sh -c "rm -rf build && cmake -B build -G Ninja && cmake --build build --target SDGTest --parallel"

- name: Run tests
  run: docker run --rm -v $(pwd):/src sd-build ctest --test-dir build/testing --output-on-failure
```

Notes:
- `rm -rf build` before configuring avoids stale cache conflicts when mounting source into container
- Building only `SDGTest` target, not the full project
- ctest runs in `build/testing/` where tests live
