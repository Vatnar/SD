To experiment a bit with networking before beginning the quest of implementing networking i took some time to learn some basic networking. To get a break from C++ and Vulkan I chose to do this in Rust. 

# Server
```rust
let cli = Cli::parse();

let server = Server::new(cli.port, cli.dir)?;
server.start_server()?;
```
Where the CLI is parsed using `clap::Parser`:
```rust
#[derive(Parser)]
struct Cli {
    #[arg(short, long, env = "FS_PORT", default_value = "9000")]
    port: u16,

    #[arg(long, env = "FS_DIR")]
    dir: PathBuf,
}
```
The server takes a directory (dir) parameter either through the environment variable or as a command line argument. The directory acts as the root of the filesystem which the server acts from. Any client can browse, download, and upload files relative to that root. But not escape that root. 
### Creating a TcpListener

```rust
let listener = TcpListener::bind(format!("127.0.0.1:{}", self.port))?;
```

# Protocol
To transmit information between the client and the server a protocol was used which was defined with protocol buffers. 

```proto
syntax = "proto3";

package protocol;

// --- Client -> Server ---

message DownloadRequest {
    string path = 1;
}

message UploadRequest {
    string path = 1;
    uint64 file_size = 2;
}

message ListRequest {
    string path = 1;
}

message Request {
    oneof command {
        DownloadRequest download = 1;
        UploadRequest upload = 2;
        ListRequest list = 3;
        FileChunk file_chunk = 4;
    }
}

// --- Server -> Client ---

message Error {
    string message = 1;
    int32 code = 2;
}

message FileChunk {
    bytes data = 1;
    bool is_last = 2;
}

message FileInfo {
    string name = 1;
    uint64 size = 2;
    bool is_directory = 3;
}

message ListResponse {
    repeated FileInfo entries = 1;
}

message Response {
    oneof payload {
        Error error = 1;
        FileChunk file_chunk = 2;
        ListResponse list = 3;
        bool ok = 4;
    }
}
```

And we a set of read and write message functions.
```rust
include!(concat!(env!("OUT_DIR"), "/protos/mod.rs"));
pub use protocol::*;

use std::io::{self, Read, Write};

pub fn write_message<W, M>(writer: &mut W, msg: &M) -> io::Result<()>
where
    W: Write,
    M: protobuf::Message,
{
    let bytes = msg
        .write_to_bytes()
        .map_err(|e| io::Error::new(io::ErrorKind::InvalidData, e))?;
    let len = bytes.len() as u32;
    writer.write_all(&len.to_be_bytes())?;
    writer.write_all(&bytes)?;
    Ok(())
}
pub fn read_message<R, M>(reader: &mut R) -> io::Result<M>
where
    R: Read,
    M: protobuf::Message,
{
    let mut len_bytes = [0u8; 4];
    reader.read_exact(&mut len_bytes)?;
    let len = u32::from_be_bytes(len_bytes) as usize;

    let mut buf = vec![0u8; len];
    reader.read_exact(&mut buf)?;

    M::parse_from_bytes(&buf)
        .map_err(|e| io::Error::new(io::ErrorKind::InvalidData, e))
}

pub const MAX_FILE_CHUNK_SIZE: usize = 64 * 1024;
pub const MAX_MESSAGE_SIZE: usize = 1024 * 1024;
```

Request are sent from the client to the server. And responses are answered from the server.
