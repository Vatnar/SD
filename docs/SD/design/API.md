The SD engine is built as a shared library with hidden symbol visibility by default. Only symbols explicitly marked with `SD_EXPORT` are part of the public API. This keeps the ABI clean and prevents internal symbols from leaking to game code.

## Visibility

In `SD/CMakeLists.txt`:

```cmake
set_target_properties(SD PROPERTIES
    CXX_VISIBILITY_PRESET hidden
    VISIBILITY_INLINES_HIDDEN ON
)
```

This hides all symbols by default. Only classes and functions annotated with `SD_EXPORT` are visible to consumers of the shared library.

## SD_EXPORT

Generated by CMake's `generate_export_header()` into `build/generated/SD/export.hpp`:

```cmake
generate_export_header(SD
    BASE_NAME SD
    EXPORT_FILE_NAME "${CMAKE_CURRENT_BINARY_DIR}/generated/SD/export.hpp"
    STATIC_DEFINE SD_STATIC_DEFINE
)
```

Usage in headers:

```cpp
class SD_EXPORT Application {
    // ...
};

SD_EXPORT void init();
```

The macro resolves to `__attribute__((visibility("default")))` on GCC/Clang and `__declspec(dllexport)` on MSVC.

## Public vs Private Headers

- **Public API** — `SD/include/SD/` is installed and available to game code.
- **Generated** — `build/generated/SD/export.hpp` is in the build interface include path and installed alongside the public headers.
- **Private/Implementation** — `SD/src/` contains `.cpp` files and any internal headers that are not installed. Game code cannot include them.

The install rules:

```cmake
install(DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}/include/" DESTINATION include)
install(FILES "${CMAKE_CURRENT_BINARY_DIR}/generated/SD/export.hpp" DESTINATION include/SD)
```

If something is not in `include/` or not marked `SD_EXPORT`, it is not part of the engine API.
